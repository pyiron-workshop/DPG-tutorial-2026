Task

Build a workflow to iterate over different (hydrostatic) strain states and get energies of the system. To break it down, we need to
  1. Construct a set of structures that span different volumes
    • Construct the initial crystal
    • Create an array of strains
    • Apply the strains
    Simpler to change the volume by altering the lattice parameter
    2. Hint: Use the IterateToDataframe node
  2. Calculate the energy at each volume
    • Loop over each strained structure
    • Run a static single-point energy calculation, so that the volume isn't changed
    • Collect the resulting energy for each structure
    3. Which node should we use? SinglePointStatic or Relax ?
  3. Gather the results into a usable format
    • Extract the volume from each structure
    • Pair each volume with its corresponding energy to form (V, E) data points
  4. Fit an equation of state (EOS) to the (V, E) data
    • Choose an EOS, using the Birch-Murnaghan equation
    • Fit the model to extract equilibrium volume V₀, cohesive energy E₀, and bulk modulus B₀
  5. Visualize the results
    • Plot the raw (V, E) data points
    • Overlay the fitted EOS curve
    • Annotate the plot with the extracted quantities

Birch-Murnaghan equation of state#

\begin{equation} E(V) = E_0 + \frac{9V_0 B_0}{16} \left{ \left[\left(\frac{V_0}{V}\right)^{2/3} - 1\right]^3 B_0’ + \left[\left(\frac{V_0}{V}\right)^{2/3} - 1\right]^2 \left[6 - 4\left(\frac{V_0}{V}\right)^{2/3}\right] \right} \end{equation}

Changing Engines#

Task: Use different Engines

  • The Ace node gives an ASE engine to evaluate the energies and forces using an ACE interatomic potential. Try using GRACE engine
  • Benchmarking means that we compare a computed property in the fit model against the "Ground truth / reality". Try the QuantumEspresso engine

⚠️ Warning: QuantumEspresso and GRACE are computationally more intensive than ACE

Before running with the QuantumEspresso or GRACE engines, please reduce the number of points in your calculation. Using a lot of points can make the workflow run very slow and may crash your Jupyter terminal.

💡 Set num_points = 7 for QuantumEspresso and num_points = 15 for GRACE in the linspace a.k.a a_list node

Several pre-fitted GRACE foundation models are available for use. You can choose from the following options:

  • MP_GRACE_1L_r6_4Nov2024

  • MP_GRACE_1L_r6_07Nov2024

  • MP_GRACE_2L_r6_11Nov2024

  • MP_GRACE_2L_r5_4Nov2024

  • MP_GRACE_2L_r5_07Nov2024

  • GRACE_2L_OAM_28Jan25

  • GRACE-1L-OAM_2Feb25

For further details about the GRACE foundation models, please refer to the Grace documentation.

Note that you only need to provide the model string as an argument—the associated potential files will be downloaded at runtime.
